判断型 Skill / For Designers Using Codex

设计师什么时候该用 Codex？

不是把 Codex 当成“生成结果的机器”，而是把它当成一个能整理混乱材料、提炼判断标准、拆解失败原因、沉淀可执行资产的工作搭子。

一句话定义

当任务进入“方向判断、素材收束、规则转译、证据验证、流程归档”的阶段，就适合交给 Codex。

  • 1它适合处理判断链条，不只处理单个结果。
  • 2它适合把审美反馈变成可复用约束。
  • 3它适合把失败样本沉淀成下一轮执行方法。

01. 先判断：这件事适不适合交给 Codex？

核心不是“Codex 能不能做”，而是“它介入后，能不能减少你的判断负担、整理成本和返工成本”。

适合交给 Codex 的任务
  • 素材很多，但方向还没有收束：截图、参考图、对话记录、产品图、网页链接混在一起。
  • 你已经有审美反馈，但还没有变成可执行语言：比如“土”“暗”“AI感强”“过年轻”“不够清爽”。
  • 需要从多个方案里提炼判断标准：哪个方向更符合人群，哪个方向和旧系列重合。
  • 需要查证工具、权限、额度、流程边界：这一步该用 Codex、ChatGPT、浏览器，还是你自己手动做。
  • 需要输出成可复用资产：HTML 提示词库、执行表、检查清单、页面结构、归档记忆。
不适合完全交给 Codex 的任务
  • 最终审美拍板：品牌调性“对不对味”，仍然需要设计师本人判断。
  • 大量图片直接生成：当你在意速度、额度、登录稳定性时，Codex 更适合写提示词和质检规则。
  • 需要私人账号复杂授权的步骤：比如网页登录、验证码、人工确认等。
  • 只靠一句“做高级一点”的任务：如果没有样张、禁区、目标人群，Codex 容易给出泛化结果。

02. 你给 Codex 的输入，不是资料堆叠，而是判断坐标

你越能提供“背景 + 样本 + 禁区 + 输出格式”，Codex 越容易从助手变成策略整理器。

A 项目背景

说明产品、平台、页面目标、销售场景、系列关系。

  • 这是什么产品？
  • 在哪个平台使用？
  • 它和旧系列有什么关系？
  • 这次真正要解决的页面问题是什么？

B 样本材料

给 Codex 看成功样本和失败样本，让它反推标准。

  • 参考图：不是照搬，而是提取元素。
  • 失败图：指出哪里不对。
  • 满意图：说明为什么满意。
  • 产品图：保证结构和颜色不跑偏。

C 约束与格式

提前告诉 Codex 什么不能做，以及最终要交付什么。

  • 目标人群和市场。
  • 不能靠近的风格。
  • 图片比例和页面模块。
  • 需要 HTML、表格、提示词库还是归档。

03. 你的追问方式：把模糊任务逼成执行系统

有效追问不是反复说“再好看一点”，而是一步步把方向、样张、变量和文件化交付压实。

1
先问方向 “你对这款壳子的策划方向怎么理解？”先看 Codex 是否理解产品定位、系列差异和人群边界。
2
再给样张 上传满意和不满意的图，不让 Codex凭空想象审美，而是从真实样本里提取规律。
3
拆控制点 把“土、暗、AI、过年轻”拆成穿搭、光线、肤质、场景、动作、构图、产品露出等变量。
4
改执行路径 如果直接生成太慢，就转成提示词库；如果浏览器卡住，就换 Chrome；如果参考图污染，就换参考策略。
5
做成资产 最后沉淀成 HTML、执行表、文件夹、归档记忆，保证下次新窗口也能接上。

04. 验证 Codex 输出时，看这 8 个点

你验证的不是它说得漂不漂亮，而是它有没有真正服务设计决策和后续执行。

✓
是否理解业务定位？有没有把产品和目标人群说准，而不是只说泛泛的高级感。
✓
是否拉开系列差异？有没有避开已经属于其他系列的情绪、色彩、人群和页面语言。
✓
是否能落到页面模块？首屏、轮播、卖点、动画、局部细节是否都有明确用途。
✓
是否能转成执行语言？有没有把审美反馈转成提示词、参考图选择和控制要点。
✓
是否说明不能做什么？禁区越清楚，AI 生成越不容易滑向旧套路。
✓
是否可复制复用？输出最好能直接复制、打开、归档，而不是只停留在聊天里。
✓
是否解释失败原因？不是只说失败，而是指出变量：参考图、肤质、光线、动作、工具路径。
✓
是否给出下一步？好的 Codex 输出会自然导向下一轮执行，而不是让你重新组织任务。

05. 失败修正公式：从“不对”到“下一轮更准”

设计师使用 Codex 的高阶能力，是把失败样本变成可控变量，而不是把失败当作一次无效尝试。

Step 五步修正

  • 保留有效部分：先确认哪些方向是对的，比如场景、动作、构图或生活事件。
  • 指出失败表现：不要只说不好看，要说暗、土、假、太年轻、太像棚拍。
  • 收窄变量：把问题拆成光线、穿搭、人物身份、参考图、产品露出、比例。
  • 改变执行路径：生成不稳就改提示词库，参考图污染就换图或洗图，浏览器卡住就换工具。
  • 沉淀成资产：把修正后的规则写进提示词库、检查清单或项目记忆。

Example 审美反馈怎么转译

“画面有点土” → 改成：穿搭需要设计感，使用低饱和但有层次的都市轻户外单品，避免廉价旅行感。

“颜色太暗” → 改成：使用明亮自然日光、清爽夏季阴影，避免深金色黄昏滤镜。

“AI感太强” → 改成：真实肤质、自然表情、脸不要主导画面、动作像生活中顺手发生。

“不像官网主视觉” → 改成：需要足够留白、清晰产品露出、页面首屏可承载标题和卖点。

06. 生图高级感控制卡：不要写“高级”，要控制变量

这段经验很适合保存成固定判断规则：高级感不是靠堆 luxury / premium，而是靠场景、身份、配色、动作和产品呈现方式共同控制。

Scene 场景降噪

场景越克制，越容易显得真实、有留白、有官网质感。不要写太多热闹元素。

  • quiet gallery street
  • minimal cafe entrance
  • creator studio
  • refined desk
  • design studio doorway

User 人物身份

不要把人物写成 model、fashion girl、influencer，这些词容易把画面带向棚拍或网红感。

  • stylish adult creator
  • design-minded user
  • face not dominant
  • real lifestyle content
  • natural presence, not posing

Color 低饱和配色

不是让画面全白，而是让环境和穿搭低饱和，手机壳成为唯一稍亮的视觉点。

  • cream
  • soft burgundy
  • muted green
  • light denim
  • warm neutral / brushed metal

Action 动作要像顺手发生

高级感很怕硬摆拍。动作应该像生活里自然发生，产品在动作里被看见。

  • checking a message while walking
  • taking a mirror outfit selfie
  • reviewing content on the phone
  • photographing coffee / table details
  • using the Ostand for hands-free content review

Product 产品不要像电商图

产品可以清楚，但要存在于真实材质和真实动作里，避免白底目录图或硬广感。

  • on a cafe table with soft shadows
  • beside a notebook and iced coffee
  • on a cream knit sleeve
  • real UGC texture, not studio product photography
  • phone case visible within 3 seconds

Template 可复用高级感生图模板

Generate a [比例，例如 1:1 / 4:5 / 16:9] photorealistic UGC-style lifestyle image.

Use the uploaded phone case reference strictly. The image should first feel like refined creator lifestyle / tasteful OOTD content, then subtly like a phone case ad.

Scene:
A quiet minimal cafe entrance, design studio doorway, gallery-adjacent street, or refined creator workspace in natural daylight. Clean but lived-in, not a studio product shoot.

Subject:
One stylish adult creator or design-minded user, face not dominant. Outfit in cream, warm neutral, soft burgundy, muted green, light denim, or tailored casual pieces. No visible brand logos.

Action:
The person is naturally using the phone: checking a message while walking, taking a mirror outfit selfie, reviewing content, or photographing coffee/table details. Avoid stiff standing or obvious modeling poses.

Phone case:
The phone case should be clearly visible and match the uploaded reference. Keep the structure, colors, camera cutout, side buttons/grip, and stand/ring details accurate. The case should feel coordinated with the outfit, not pasted on.

Lighting:
Soft natural daylight, gentle shadows, real material texture, slightly imperfect UGC realism.

Avoid:
poster, infographic, text overlay, fake logos, celebrity likeness, watermark, distorted phone case, wrong colors, distorted hands, extra fingers, product catalog style, studio white background, overly glossy fashion campaign look.

高级感 = 低饱和配色 + 安静真实场景 + creator 身份 + 顺手动作 + 产品细节准确。

这条规则适合写进所有生活方式类产品生图提示词，尤其适合避免“土、假、棚拍、网红感”。

07. 可直接复用的提问模板

以后做页面策划、AI 生图、视觉方向、提示词库，都可以用这套句式开局。

Brief 给 Codex 的开场

适合你要它先理解项目，而不是直接动手时使用。

我现在有一个设计项目，需要你先帮我判断方向。
项目背景是：[产品 / 平台 / 目标页面]。
我已有的素材在：[文件夹 / 截图 / 链接]。
目前我认为方向可能是：[初步判断]，但你可以反驳。
请你不要直接总结素材，而是帮我判断：目标人群、差异化、视觉禁区、页面表达重点，以及后续执行方式。

Review 给 Codex 的修正

适合你已经有样张，需要它把反馈变成下一轮规则时使用。

我认可的部分是：[哪里对]。
目前不满意的是：[具体问题]。
请你不要只说“优化”，而是拆成：场景、人物、穿搭、光线、构图、产品露出、参考图使用、负面约束。
最后输出成：[执行表 / 提示词库 / HTML / 检查清单]，方便我下一轮直接使用。

Codex 最适合处理的不是“我要一个结果”，而是“我需要把混乱材料变成判断系统”。

设计师保留最终审美判断，Codex 负责把判断过程结构化、执行化、资产化。
